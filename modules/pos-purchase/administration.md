# Administration

Manage user accounts and system access.

## Overview

Administration allows you to create and manage user accounts for tenants and staff.

## Accessing Administration

Navigate to **Administration** from the main menu or **POS Purchase > Administration**

## Creating Tenant User Login

### Step 1: Access Create User

Navigate to **Administration > Create User**

### Step 2: Select User Type

| Type | Description |
|------|-------------|
| **Admin** | Full system access |
| **Manager** | Management functions |
| **Merchant** | Tenant/store access |
| **Staff** | Limited operational access |

### Step 3: Enter User Details

| Field | Description |
|-------|-------------|
| **User Email** | Login email |
| **Username** | System username |
| **First Name** | User's first name |
| **Last Name** | User's last name |
| **Mobile No** | Contact number |
| **NRIC** | ID number |
| **Type** | User role |
| **Merchant** | Assigned tenant (if merchant type) |
| **Gender** | User's gender |
| **Password** | Initial password |
| **Retype Password** | Confirm password |

### Step 4: Create

Click **Create** to generate the user account.

## User Types and Permissions

### Admin
- Full system access
- All configuration options
- All reports

### Manager
- Operational management
- Limited configuration
- Relevant reports

### Merchant
- Own tenant data only
- Transaction processing
- Tenant reports

### Staff
- Basic operations
- No configuration
- Limited data access

## Managing Users

### Edit User
1. Find user in list
2. Click Edit
3. Modify details
4. Save

### Deactivate User
1. Find user
2. Change status to Inactive
3. Save

### Reset Password
1. Find user
2. Click Reset Password
3. New password sent to user

## Best Practices

{% hint style="tip" %}
**User Management:**
- Use descriptive usernames
- Assign appropriate roles
- Review access regularly
- Deactivate departed users promptly
{% endhint %}

{% hint style="warning" %}
**Security:**
- Enforce strong passwords
- Regular access reviews
- Log all admin activities
{% endhint %}

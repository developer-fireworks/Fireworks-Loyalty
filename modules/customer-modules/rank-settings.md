# Rank Settings

Learn how to configure membership tiers and ranks in the Loyalty CRM system.

## Overview

Rank Settings allow you to define and manage membership tiers for your loyalty program, creating a tiered structure that rewards customer engagement.

## Accessing Rank Settings

Navigate to **Customer > Rank Settings**

<figure><img src="../../.gitbook/assets/unknown (1).png" alt=""><figcaption></figcaption></figure>

## Understanding Ranks

Ranks (or Tiers) represent different levels of membership status, typically based on:

* Points accumulated
* Spending thresholds
* Membership duration
* Activity levels

## Common Tier Structures

| Tier             | Typical Requirements | Benefits                        |
| ---------------- | -------------------- | ------------------------------- |
| **Basic/Bronze** | Entry level          | Standard earning rate           |
| **Silver**       | Mid-level engagement | Enhanced earning, perks         |
| **Gold**         | High engagement      | Premium benefits                |
| **Platinum**     | Top tier             | VIP treatment, exclusive access |

## Rank Configuration Options

Each rank can be configured with:

| Setting              | Description                        |
| -------------------- | ---------------------------------- |
| **Rank Name**        | Display name for the tier          |
| **Minimum Points**   | Points required to achieve tier    |
| **Point Multiplier** | Earning rate multiplier            |
| **Benefits**         | Tier-specific rewards and perks    |
| **Validity Period**  | How long tier status is maintained |
| **Upgrade Rules**    | Conditions for tier promotion      |
| **Downgrade Rules**  | Conditions for tier demotion       |

## Rank List View

The rank list displays:

* All configured tiers
* Current member count per tier
* Rank requirements
* Action options (Edit/Delete)

## Best Practices

{% hint style="info" %}
**Tier Design:**

* Keep tier names simple and aspirational
* Set achievable yet meaningful thresholds
* Ensure clear differentiation between tiers
* Communicate benefits clearly to members
{% endhint %}

{% hint style="info" %}
**Balance Accessibility:** Make lower tiers achievable for most customers while keeping top tiers exclusive enough to feel special.
{% endhint %}

{% hint style="warning" %}
**Changing Requirements:** Be cautious when modifying tier requirements as it may affect existing member statuses. Communicate changes to affected members.
{% endhint %}

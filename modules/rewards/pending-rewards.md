# Pending Rewards

Process and approve pending reward redemptions.

## Overview

Pending Rewards shows all redemption requests awaiting approval or processing.

## Accessing Pending Rewards

Navigate to **Rewards Module > Pending Rewards**

## Pending Rewards List

| Column | Description |
|--------|-------------|
| Request ID | Unique identifier |
| Member | Requesting member |
| Reward | Requested item |
| Points | Points deducted |
| Date | Request date |
| Status | Pending/Processing |
| Actions | Approve/Reject |

## Approving a Reward

### Step 1: Review Request

Click on the pending request to view details:
- Member information
- Reward details
- Points transaction
- Request notes

### Step 2: Approve or Reject

| Action | Description |
|--------|-------------|
| **Approve** | Confirm redemption |
| **Reject** | Deny redemption (points refunded) |
| **Request Info** | Ask for additional information |

### Step 3: Process Fulfillment

For approved rewards:
- Physical: Prepare for pickup/delivery
- Digital: Generate/send voucher code
- Experience: Confirm booking

### Step 4: Mark Complete

Update status to Completed once fulfilled.

## Bulk Processing

For multiple requests:
1. Select requests using checkboxes
2. Choose bulk action (Approve All/Reject All)
3. Confirm action

## Filtering Pending Rewards

Filter by:
- Date range
- Reward category
- Member tier
- Location
- Status

## Best Practices

{% hint style="tip" %}
**Timely Processing:**
- Process requests daily
- Set SLA for approvals
- Communicate delays to members
{% endhint %}

{% hint style="info" %}
**Verification:**
- Verify member identity for pickup
- Confirm contact details for delivery
- Document handover
{% endhint %}

{% hint style="warning" %}
**Rejection Handling:**
Always provide clear reasons for rejections and ensure points are properly refunded.
{% endhint %}

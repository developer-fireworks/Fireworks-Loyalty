# Category Management

Organize rewards into categories for better navigation and management.

## Overview

Category Management allows you to create logical groupings for your rewards, making it easier for members to browse and find desired items.

## Accessing Category Management

Navigate to **Rewards Module > Category Management**

## Category List View

| Column | Description |
|--------|-------------|
| Category Name | Display name |
| Code | Unique identifier |
| Rewards Count | Items in category |
| Status | Active/Inactive |
| Actions | Edit/Delete |

## Adding a Category

### Step 1: Click Add Category

Navigate to **Category Management > Add Category**

### Step 2: Enter Category Details

| Field | Description | Required |
|-------|-------------|----------|
| **Category Name** | Display name | Yes |
| **Category Code** | Unique identifier | Yes |
| **Description** | Category description | No |
| **Parent Category** | Nest under another | No |
| **Image** | Category thumbnail | No |
| **Status** | Active/Inactive | Yes |
| **Sort Order** | Display sequence | No |

### Step 3: Save Category

Click **Save** to create the category.

## Editing a Category

### Step 1: Access Edit

Click **Action > Edit** on the category.

### Step 2: Modify Details

Update any fields as needed.

### Step 3: Save Changes

Click **Save** to apply changes.

## Deleting a Category

### Step 1: Initiate Delete

Click **Action > Delete** on the category.

### Step 2: Confirm Deletion

Confirm the deletion when prompted.

{% hint style="danger" %}
**Warning:** Deleting a category does not delete rewards within it, but rewards will become uncategorized. Reassign rewards before deleting.
{% endhint %}

## Category Hierarchy

Create parent-child relationships:

```
Electronics (Parent)
├── Gadgets (Child)

└── Audio (Child)

Vouchers (Parent)

```

## Best Practices

{% hint style="tip" %}
**Clear Naming:**
- Use intuitive category names
- Keep hierarchy simple (max 2-3 levels)
- Consider member perspective
{% endhint %}

{% hint style="info" %}
**Organization:**
- Group related rewards logically
- Balance category sizes
- Avoid too many or too few categories
{% endhint %}

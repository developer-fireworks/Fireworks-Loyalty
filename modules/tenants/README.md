# Tenants Overview

Learn how to manage tenants and merchants in the Loyalty CRM system.

## Overview

The Tenant Management module allows administrators to create and manage merchant accounts that participate in the loyalty program.

## Accessing Tenant Management

Navigate to **Tenants > Tenant Management**

<figure><img src="../../.gitbook/assets/unknown (13).png" alt=""><figcaption></figcaption></figure>

## What is a Tenant?

A tenant (or merchant) represents a business entity that:

* Participates in the loyalty program
* Issues and redeems points
* Offers rewards to members
* Has access to merchant-specific features

## Tenant List View

The tenant list displays:

| Column      | Description            |
| ----------- | ---------------------- |
| Tenant Name | Business name          |
| Tenant Code | Unique identifier      |
| Category    | Business category      |
| Status      | Active/Inactive        |
| Contact     | Primary contact person |
| Actions     | Edit/Delete options    |

## Key Features

### Tenant Capabilities

* Point issuance and redemption
* Reward management
* Transaction processing
* Performance reporting
* Customer data access (within scope)

### Tenant Configuration

* Business information
* Contact details
* Operating hours
* Location details
* Commission settings

## Best Practices

{% hint style="info" %}
**Onboarding Tenants:**

1. Verify business documentation
2. Configure appropriate permissions
3. Provide training materials
4. Set up communication channels
{% endhint %}

{% hint style="info" %}
**Tenant Categories:** Organize tenants by category (F\&B, Retail, Services, etc.) for better management and reporting.
{% endhint %}

{% hint style="warning" %}
**Access Control:** Ensure tenant accounts have appropriate permission levels - merchants should only access their own data.
{% endhint %}
